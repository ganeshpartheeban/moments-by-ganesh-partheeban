01 · Goals

  • Fast, content-first marketing site.
  • Mobile-first — most traffic on phones.
  • SEO-optimised for German + English queries.
  • Lead capture without a backend server.
  • Cloudflare-hosted, GitHub Actions auto-deploy.
  • Lighthouse 95+ across the board on mobile.

02 · Tech stack (fix this stack — do not propose alternatives)

  • Framework: TanStack Start (SSR React 19) with file-based routing via TanStack Router. Use typed routes and dynamic params (e.g. /work/$slug).
  • Bundler: Vite 7.
  • Language: TypeScript, strict mode.
  • Styling: Tailwind CSS 4 with the OKLCH colour system. No CSS-in-JS.
  • Hosting: Cloudflare Workers via @cloudflare/vite-plugin. wrangler.jsonc for config. Don't propose Pages or Vercel.
  • Deploy: GitHub Actions calling cloudflare/wrangler-action@v3. Pass build-time secrets via env: and Cloudflare deploy secrets via with:.
  • Image processing: sharp for variant generation, exifr for EXIF read at build time. Generate AVIF + WebP + JPEG at multiple widths (e.g. 480/600/800/1000/1400) for thumbs, plus a full-res variant for the lightbox.
  • Forms: Google Apps Script as the backend (no server). Token-gated POST endpoint, URL + token injected via VITE_FORM_ENDPOINT and VITE_FORM_TOKEN env vars. Submissions write to Google Sheets + trigger email notifications. Add a graceful 3-provider IP geolocation lookup (ipapi.co → ipwho.is → api.country.is, 4s timeout each), fired lazily on first form focus.
  • Analytics: Cloudflare Web Analytics (free, no cookies). Bind via the Cloudflare dashboard at the edge — no script tag needed.
  • Fonts: self-host one editorial serif (Fraunces variable) + one geometric sans (Inter Tight variable) via @fontsource-variable. Use font-display: swap. If using Google Fonts CDN, load with media="print" onLoad="this.media='all'" so it never blocks render.

04 · SEO requirements

Emit JSON-LD on every relevant route — schemas to implement:

  • Organization (@id set, single source) — root.
  • Person (founder) — root.
  • ProfessionalService or Store depending on offering — root.
  • BreadcrumbList — every secondary route.
  • ImageGallery with ImageObject entries that include caption, contentLocation, dateCreated — galleries and /work/$slug.
  • Review + AggregateRating wrapped under the org @id — page with testimonials. Use real customer quotes from the PDF.
  • FAQPage — services / FAQ section if present.

Per-page OG images:

  • One generic /og-hero.jpg (1200×630) at root.
  • Per-story OG images generated at build time via scripts/generate-og.mjs using sharp to smart-crop each story's cover to 1200×630 and write to /public/og/<slug>.jpg. Wire as prebuild in package.json.
  • Every page emits og:title, og:description, og:image (with width/height/alt/type), twitter:card=summary_large_image, twitter:image.

Other SEO:

  • <link rel="canonical"> on every page.
  • robots: index, follow, max-image-preview:large.
  • og:locale=de_DE primary; alternate en_GB.
  • <html lang="de"> (or current locale, dynamic).
  • Self-hosted favicon + apple-touch-icon + brand logo as both favicon source and OG fallback.

05 · i18n

Context-based i18n with useI18n() hook. Strings live in a single src/lib/i18n.tsx module as Record<Locale, Record<Key, string>>.

  • Locales: de (primary), en. Optionally fr, es, it — leave scaffolding so adding one is purely a strings file.
  • Toggle stored in localStorage. Default locale auto-detected from navigator.language on first visit, fallback de.
  • Language toggle built but optionally hidden behind a feature flag if launching with one locale only.

06 · Performance non-negotiables

  • Code-splitting: React.lazy + Suspense for the lightbox, modals, cookie notice, floating contact button, bottom tab bar.
  • Hero LCP: <link rel="preload" as="image" imagesrcset imagesizes> with a 480w variant for mobile. fetchpriority="high" on the hero img.
  • Content-visibility: content-visibility: auto + contain-intrinsic-size on off-screen masonry photos.
  • IntersectionObserver: pause expensive animations when offscreen.
  • matchMedia, not resize listeners, for responsive column counts.
  • Async font CSS: never render-blocking.
  • Image sizes attr: every gallery img has a correct sizes= to avoid over-fetching on mobile.
  • Sitemap, robots.txt, well-formed cache headers via Cloudflare _headers file.

07 · Mobile UX

  • Bottom tab bar: 4-tab fixed-bottom nav. Hide on scroll-down, show on scroll-up. env(safe-area-inset-bottom) padding for notched devices.
  • Header: sticky top with brand on the left, Enquire CTA on the right (visible on mobile). No hamburger — bottom tab bar replaces it.
  • Floating WhatsApp (source-aware): prefilled message includes context from current route. Sits above the bottom tab bar.
  • Tap feedback: active:scale-[0.985] with transition-transform on every photo/card button, motion-reduce:transition-none.
  • Page transitions: 220ms cross-fade on route change via <main key={pathname} className="page-transition"> and a CSS keyframe. Respect prefers-reduced-motion.
  • Scroll-to-top on route change (subscribe to router onResolved, skip when to.hash is present).
  • overflow-x: clip on html/body as a safety net.
  • Image lightbox: full-screen, keyboard + swipe nav, LQIP blur placeholder (start at filter: blur(14px), clear on onLoad), photoIndices prop to scope the cycler to a subset, position indicator (n / total), safe-area-inset padding.
  • Cookie notice: 10s auto-dismiss with persistent flag, safe-area aware.
  • Engagement modal: 1-day cooldown in localStorage.

08 · Content patterns reused

  • Per-photo metadata module (src/lib/photo-meta.ts) keyed by 1-based image number, with title/detail/venue/location/date/featuring. Used for alt text, gallery captions, lightbox captions, and ImageGallery JSON-LD.
  • Case-study / story module (src/lib/case-studies.ts) with slug, title, location, date, kind, intro, cover, sequence — driving the dynamic /work/$slug route and the home story grid.
  • Animated StoryCards that cycle through their sequence on a stagger (per-card delay + drift) with lazy-mounted frames so initial DOM weight stays low.
  • Footer Index No. that changes per route (01 · Home, 02 · About, Story · <Title>) for an editorial touch.

e · Apps Script source — paste into Code.gs

Reconstructed from the actual form contract used by this client. Drop-in compatible.

// Inkyhaus enquiry intake — Google Apps Script Web App.
// Append form submissions to a Google Sheet and notify the owner by email.

const FORM_TOKEN = "REPLACE_WITH_SAME_VALUE_AS_VITE_FORM_TOKEN";
const SHEET_ID = "REPLACE_WITH_GOOGLE_SHEET_ID";       // from the sheet URL
const SHEET_NAME = "Enquiries";                          // tab name
const NOTIFY_EMAIL = "you@inkyhaus.de";                 // owner inbox

const HEADERS = [
  "Timestamp", "Type", "Name", "Email", "Phone",
  "EventType", "Date", "Location", "Budget", "Message",
  "Source", "IPLocation", "UserAgent",
];

function doPost(e) {
  try {
    const params = (e && e.parameter) || {};
    if (params.token !== FORM_TOKEN) {
      return _json({ ok: false, error: "Invalid token" }, 403);
    }

    const sheet = _openSheet();
    const now = new Date();
    const row = [
      now,
      params.type || "enquiry",
      params.name || "",
      params.email || "",
      params.phone || "",
      params.eventType || "",
      params.date || "",
      params.location || "",
      params.budget || "",
      params.message || "",
      params.source || "",
      params.ipLocation || params.location || "",
      params.userAgent || "",
    ];
    sheet.appendRow(row);

    _notify(params, now);

    return _json({ ok: true });
  } catch (err) {
    return _json({ ok: false, error: String(err) }, 500);
  }
}

function doGet() {
  return _json({ ok: true, service: "Inkyhaus enquiry intake" });
}

function _openSheet() {
  const ss = SpreadsheetApp.openById(SHEET_ID);
  let sheet = ss.getSheetByName(SHEET_NAME);
  if (!sheet) {
    sheet = ss.insertSheet(SHEET_NAME);
    sheet.appendRow(HEADERS);
    sheet.setFrozenRows(1);
  } else if (sheet.getLastRow() === 0) {
    sheet.appendRow(HEADERS);
    sheet.setFrozenRows(1);
  }
  return sheet;
}

function _notify(p, when) {
  const subject = `New enquiry · ${p.name || "anon"} · ${p.eventType || p.type || ""}`;
  const body = [
    `Time: ${when.toISOString()}`,
    `Name: ${p.name || "-"}`,
    `Email: ${p.email || "-"}`,
    `Phone: ${p.phone || "-"}`,
    `Type: ${p.eventType || p.type || "-"}`,
    `Date: ${p.date || "-"}`,
    `Location: ${p.location || "-"}`,
    `Budget: ${p.budget || "-"}`,
    `Source: ${p.source || "-"}`,
    `IP location: ${p.ipLocation || p.location || "-"}`,
    "",
    "Message:",
    p.message || "(none)",
  ].join("\n");
  MailApp.sendEmail(NOTIFY_EMAIL, subject, body);
}

function _json(payload, statusCode) {
  // Apps Script Web Apps can't set arbitrary status codes via ContentService,
  // but the client uses mode:"no-cors" anyway — payload is for debugging.
  if (statusCode) Logger.log("status=" + statusCode + " body=" + JSON.stringify(payload));
  return ContentService
    .createTextOutput(JSON.stringify(payload))
    .setMimeType(ContentService.MimeType.JSON);
}

Deploy steps for the Apps Script:

  1. Create a new Google Sheet → copy its ID from the URL.
  2. Extensions → Apps Script → paste the code above.
  3. Fill in FORM_TOKEN, SHEET_ID, NOTIFY_EMAIL.
  4. Deploy → New deployment → type: Web app → Execute as: Me, Who has access: Anyone → Deploy.
  5. Copy the resulting /exec URL into VITE_FORM_ENDPOINT.
  6. The form uses mode: "no-cors" so the browser can't read the response — the script logs and emails the owner so you have a delivery trail.

f · Client-side form submission pattern

const handleSubmit = async (e: React.FormEvent<HTMLFormElement>) => {
  e.preventDefault();
  const form = e.currentTarget;
  const data = new FormData(form);
  data.set("token", FORM_TOKEN);
  data.set("type", "enquiry");
  data.set("source", window.location.pathname);
  data.set("userAgent", navigator.userAgent);
  // Optionally inject IP-geo lookup result here.
  await fetch(FORM_ENDPOINT, {
    method: "POST",
    body: data,
    mode: "no-cors",
  });
};

g · IP geolocation helper

Three-provider fallback with per-request timeout. Resolves to a human-readable string like Berlin, Berlin, Germany (94.x.x.x) or empty if every provider is blocked. Fire it lazily on first form focus so it doesn't burn requests on every page view.

const LOCATION_TIMEOUT_MS = 4000;

function withTimeout<T>(promise: Promise<T>, ms: number): Promise<T> {
  return new Promise((resolve, reject) => {
    const id = setTimeout(() => reject(new Error("timeout")), ms);
    promise.then(
      (v) => {
        clearTimeout(id);
        resolve(v);
      },
      (err) => {
        clearTimeout(id);
        reject(err);
      },
    );
  });
}

async function resolveLocation(): Promise<string> {
  const providers: Array<() => Promise<string>> = [
    async () => {
      const r = await fetch("https://ipapi.co/json/");
      const j = await r.json();
      const parts = [j.city, j.region, j.country_name].filter(Boolean);
      return parts.length ? `${parts.join(", ")}${j.ip ? ` (${j.ip})` : ""}` : "";
    },
    async () => {
      const r = await fetch("https://ipwho.is/");
      const j = await r.json();
      if (j && j.success !== false) {
        const parts = [j.city, j.region, j.country].filter(Boolean);
        return parts.length ? `${parts.join(", ")}${j.ip ? ` (${j.ip})` : ""}` : "";
      }
      return "";
    },
    async () => {
      const r = await fetch("https://api.country.is/");
      const j = await r.json();
      return j && j.country ? `${j.country}${j.ip ? ` (${j.ip})` : ""}` : "";
    },
  ];
  for (const p of providers) {
    try {
      const result = await withTimeout(p(), LOCATION_TIMEOUT_MS);
      if (result) return result;
    } catch {
      // try next provider
    }
  }
  return "";
}

Wiring (deferred trigger):

const locationRef = useRef<string>("");
const locationPromiseRef = useRef<Promise<void> | null>(null);

const ensureLocationLookup = () => {
  if (locationPromiseRef.current) return;
  locationPromiseRef.current = resolveLocation()
    .then((loc) => {
      locationRef.current = loc;
    })
    .catch(() => {
      // All providers blocked / offline — submit goes through with empty location.
    });
};

// On the form root:
<form onFocusCapture={ensureLocationLookup} onSubmit={handleSubmit}>...</form>

// Inside handleSubmit, wait up to 2s before sending:
if (!locationRef.current && locationPromiseRef.current) {
  try { await withTimeout(locationPromiseRef.current, 2000); } catch {}
}
data.set("location", locationRef.current);

h · Rate-limit-per-email pattern

localStorage-based soft cap so a single email can't spam the inbox. Apps Script-side validation should still exist for the real ceiling (this only blocks the casual repeat-clicker).

const SUBMISSIONS_KEY = "inkyhaus-enquiry-emails";
const MAX_PER_EMAIL = 5;

function getSubmittedCount(email: string): number {
  try {
    const raw = localStorage.getItem(SUBMISSIONS_KEY);
    const map: Record<string, number> = raw ? JSON.parse(raw) : {};
    return map[email.toLowerCase().trim()] ?? 0;
  } catch {
    return 0;
  }
}

function recordSubmission(email: string) {
  try {
    const raw = localStorage.getItem(SUBMISSIONS_KEY);
    const map: Record<string, number> = raw ? JSON.parse(raw) : {};
    const key = email.toLowerCase().trim();
    map[key] = (map[key] ?? 0) + 1;
    localStorage.setItem(SUBMISSIONS_KEY, JSON.stringify(map));
  } catch {
    // localStorage may be unavailable (private mode, storage quota).
    // Server-side validation in the Apps Script remains the real ceiling.
  }
}

Wiring inside handleSubmit:

const email = String(data.get("email") || "").trim();
if (getSubmittedCount(email) >= MAX_PER_EMAIL) {
  setStatus("limit");                // show a friendly "you've already enquired" message
  return;
}
// ... send ...
recordSubmission(email);             // only on successful POST
Server-side hardening (optional): add a similar counter inside the Apps Script keyed by email using PropertiesService.getScriptProperties() — that survives across browsers and incognito, while the localStorage version only stops the same device.
